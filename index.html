1. Yêu cầu & khởi chạy

  • Yêu cầu hệ thống: .NET Runtime 8 (hoặc Visual Studio 2022 / 2023 với .NET 8 SDK) và kết nối đến SQL Server chứa database dự án.
  • Cách chạy:
    • Mở solution bằng Visual Studio → Build → Run (F5) hoặc chạy file exe trong thư mục output.
    • File cấu hình kết nối nằm trong lớp DbContext. Mặc định dự án dùng SQL Server (xem file kết nối trong code).
  • Quyền: tài khoản đăng nhập có phân quyền (Thu ngân / Kho / Quản lý) để hiện - ẩn các nút nghiệp vụ.

2. Đăng nhập & Quên mật khẩu

Trang đăng nhập yêu cầu Tên đăng nhập và Mật khẩu. Sau khi đăng nhập thành công, hệ thống sẽ gán thôngত্তি người dùng (tên, quyền, chi nhánh) vào giao diện chính.

  • Quên mật khẩu: dùng liên kết "Quên mật khẩu" để mở form lấy lại mật khẩu (theo luồng đã triển khai).
  • Nếu tài khoản bị khóa (TrangThai = false) sẽ không thể đăng nhập.

3. Giao diện & điều hướng chính

Thanh bên trái chứa menu chức năng chính: Dashboard, Đơn hàng, Sản phẩm, Khách hàng, Nhập hàng, Thống kê, Quản lý, Cài đặt, Đăng xuất.

Phần giữa là khu vực làm việc (UserControl) — các module sẽ được load vào panel chính.

Phần phải hiển thị thông tin người dùng và banner ảnh. Avatar người dùng lấy từ dữ liệu trong CSDL.

4. Dashboard (Trang tổng quan)

Hiển thị các chỉ số nhanh: tổng đơn, doanh thu, số sản phẩm, biểu đồ top sản phẩm bán chạy. Trang tự động load dữ liệu khi mở.

5. Đơn hàng

Danh sách & thao tác

  • Hiển thị danh sách hóa đơn, bạn có thể tìm kiếm theo nhiều tiêu chí (Mã, Tên khách, Điện thoại, Thu ngân, Thanh toán, Tổng tiền, Ngày bán).
  • Chức năng: Tạo hóa đơn, Sửa, Xóa (xóa sẽ hoàn trả hàng vào kho), In hóa đơn, Xem chi tiết.
  • Import/Export: có chức năng export dữ liệu thô dưới dạng CSV (dùng để backup) và import trở lại (cần đúng định dạng).

6. Sản phẩm

Module quản lý sản phẩm bao gồm danh sách dạng thẻ (ProductCard), thêm/sửa/xóa, tìm kiếm, lọc, sắp xếp.

Một số lưu ý quan trọng

  • ProductCard hiển thị theo thiết kế file Designer của control UC_ProductCard — mặc định hiển thị font kích thước thiết kế (thiết kế ban đầu), thay đổi kích thước hệ thống KHÔNG làm thay đổi font hiển thị ProductCard (điều này là intentional).
  • Nếu bạn muốn ProductCard đổi theo cài đặt hệ thống, hãy chỉnh UC_ProductCard (labels) để chịu ảnh hưởng font chung hoặc bật tính năng đồng bộ trong mã.
  • Để tránh lặp card khi reload: hệ thống đã xóa sạch controls trước khi add lại; nếu vẫn gặp lặp, kiểm tra log hoặc gọi Load lại (F5).

7. Khách hàng

Quản lý thông tin khách hàng: thêm, sửa, tìm kiếm. Các thông tin bao gồm tên, điện thoại, điểm tích lũy, địa chỉ.

8. Backup / Import (CSV)

Trong module Đơn hàng có chức năng xuất file CSV backup. Định dạng file xuất gồm các cột thô phục vụ import lại.

Import chặt chẽ: nếu dữ liệu trùng hoặc tham chiếu tới mã không tồn tại (khách hàng, nhân viên, sản phẩm, hình thức thanh toán) thì quá trình import sẽ bị từ chối để tránh dữ liệu hỏng.

9. Báo cáo & In ấn

Ứng dụng hỗ trợ báo cáo doanh thu (ReportViewer) và in hóa đơn. Báo cáo có thể cấu hình bộ lọc (từ ngày, đến ngày, nhân viên, ngưỡng doanh thu).

10. Quản lý (Admin)

Nếu bạn có quyền Quản lý/Admin, hiển thị thêm module Quản lý: Nhân viên, Nhà cung cấp, Khuyến mãi, Chi nhánh, Hình thức thanh toán,...

Phân quyền được đọc từ trường chức vụ/tên quyền (cập nhật tại thời điểm đăng nhập).

11. Cài đặt giao diện & font

Vào Cài đặt → Giao diện/Hiển thị để thay đổi:

  • Theme (Dark, Blue, Green, ...)
  • Font hệ thống
  • Kích cỡ chữ (hệ thống) — hợp lệ: 9 → 16
Lưu ý thiết kế:
  • Thay đổi font/kích cỡ sẽ được áp dụng đệ quy lên hầu hết các control trên form chính và usercontrols, nhưng ProductCard đã được cấu hình giữ nguyên font thiết kế để bảo toàn bố cục thẻ. Nếu bạn muốn ProductCard cũng thay đổi, cần sửa UC_ProductCard để cho phép đồng bộ.

12. Hồ sơ cá nhân & Đổi mật khẩu

  • Trong Cài đặt phần "Hồ sơ" bạn có thể cập nhật: tên hiển thị, số điện thoại, email, ngày sinh, giới tính và avatar.
  • Đổi mật khẩu: mở form đổi mật khẩu, nhập mật khẩu cũ và mật khẩu mới, hệ thống kiểm tra khớp và lưu xuống DB.
  • Quên mật khẩu: sử dụng form Quên mật khẩu để lấy lại theo luồng đã triển khai.

13. Phím tắt quan trọng

  • F2 — Tạo mới (thường ở các module: sản phẩm / hóa đơn)
  • F4 — Sửa mục chọn
  • Delete — Xóa (khi có quyền)
  • F3 — Focus ô tìm kiếm
  • F5 — Tải lại dữ liệu hiện tại

14. Các lỗi thường gặp & khắc phục

Không khởi động / lỗi kết nối DB

Kiểm tra connection string trong cấu hình DbContext, đảm bảo SQL Server đang chạy và user có quyền truy cập.

Avatar/ảnh banner không hiển thị

  • Kiểm tra trường ảnh trong DB (blob) được lưu đúng định dạng PNG/JPEG.
  • Nếu resource mặc định bị xóa, phục hồi file trong Properties/Resources hoặc đặt ảnh mặc định mới.

ProductCard không đổi font khi thay theme

Đây là hành vi cố ý: ProductCard giữ font từ thiết kế. Nếu muốn thay đổi, chỉnh file UC_ProductCard để cho phép nhận font hệ thống.

15. Câu hỏi thường gặp (FAQ)

Tôi là nhân viên, không thấy nút Xóa?

Hệ thống ẩn các chức năng nhạy cảm nếu tài khoản không thuộc nhóm Quản lý/Admin.

Muốn thêm trường cho sản phẩm (ví dụ: màu sắc) phải làm sao?

Sửa model SanPham trong thư mục Data, cập nhật DbContext/migrations và chỉnh UI tương ứng (form thêm/sửa sản phẩm + hiển thị trên ProductCard).

16. Liên hệ & Hỗ trợ

Nếu cần hỗ trợ triển khai/tuỳ biến, liên hệ với người phát triển dự án hoặc quản lý codebase. Ghi chú thông tin:

  • Chi tiết lỗi (screenshot, stack trace nếu có)
  • Phiên bản .NET và hệ điều hành
  • Hướng dẫn phục hồi DB nếu cần

Ghi chú: tài liệu này soạn dựa trên mã nguồn hiện có. Nếu bạn chỉnh sửa cấu trúc control hoặc logic, cập nhật hướng dẫn tương ứng.
